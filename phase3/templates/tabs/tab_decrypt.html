⚙️ Decryption Pipeline — Live Steps

Watch the server work through each step. Notice that tag verification happens before any decryption attempt.
📥 Step 1 — Receive Nonce + Ciphertext + Tag Waiting
The server receives all three required values from the client.
🔏 Step 2 — Recompute GHASH Tag Waiting
The server independently recomputes the GHASH authentication tag over the received ciphertext using the secret key. Decryption has not started yet.
✅ Step 3 — Compare Tags (Constant-Time) Waiting
The computed tag is compared to the received tag in constant time to prevent timing attacks. If they do not match, the request is rejected here — no decryption, no error details, no oracle.
🔓 Step 4 — AES-256 Decryption (CTR mode) Waiting
Only reached if the tag matched. The server decrypts using AES-CTR mode with the nonce and secret key. No padding to check — no padding oracle possible.
📤 Step 5 — Return Result Waiting
The plaintext is returned to the client.

🎓 What This Tab Proves
By using this tab you can see firsthand that:

1. The attacker never needs the AES key — only IV + Ciphertext.
2. The server willingly decrypts attacker-supplied ciphertext and reveals padding validity through its HTTP response code.
3. By brute-forcing one byte position, you can find the correct guess that produces valid padding — the exact oracle signal the automated attack exploits.

Scale this to all 16 bytes × all blocks = full plaintext recovery. That is the entire attack.
